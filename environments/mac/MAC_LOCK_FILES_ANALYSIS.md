## Problem Analysis

### Current Lock Files Issue
The existing [`conda-lock.txt`](conda-lock.txt) and [`requirements-lock.txt`](requirements-lock.txt) were generated FROM a Mac `sip-lims` environment but FOR Docker Linux containers, making them unsuitable for native Mac distribution.

**Evidence:**
- [`conda-lock.txt`](conda-lock.txt:3) shows `# platform: linux-aarch64`
- All conda package URLs contain `linux-aarch64` paths
- Generated by [`build/generate_lock_files.sh`](build/generate_lock_files.sh:17-21) which extracts from Docker containers

### Docker Build Process Analysis
The [`build/generate_lock_files.sh`](build/generate_lock_files.sh) script:
1. Runs `conda list --explicit` inside a Linux Docker container
2. Runs `pip freeze` inside the same Linux container
3. This converts Mac packages to their Linux equivalents
